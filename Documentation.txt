There should be 6 database schema named as:
- customerDetails
- casaAccountDetails
- loanAccountDetails
- tdAccountDetails
- rdAccountDetails
- transactions
- otp

----------------------------------------------------------------------

In customerDetails, the fields will be like this:
- customer_id (system generated, primary key)
- first_name
- middle_name
- last_name
- contact_number
- email_address
- profile_img (file path)
- accounts{
    casa: {
        account_number: casa_account_number (taken from casaAccountDetails table )
        balance:
        debit_card_number: (unique and autogenerated)
        debit_card_varient: (debit_card_varient taken from casaAccountDetails table as a foreign key, both data should be same)
        currency: (currency taken from casaAccountDetails table as a foreign key, both data should be same)
        }
    loan: {
        account_number: loan_account_nubmer (taken from loanAccountDetails table )
        balance: (loan_amount taken from loanAccountDetails table as a foreign key, both data should be same)
        }
    td: {
        account_number: td_account_number (taken from tdAccountDetails table )
        balance: (deposit_amount taken from tdAccountDetails table and directly the value is put, can or cannot be same as in tdAccountDetails table )
        }
    rd: {
        account_number: rd_account_number (taken from rdAccountDetails table )
        balance: 0 (Should be stored as 0 when the rd account is created)
        }
}

In transactions, the fields will be like this:

- transaction_id (system generated, primary key)
- customer_id (foreign key from customerDetails)
- transaction_type (enum: credit, debit)
- transaction_amount
- currency
- transfer_customer_id
- transaction_date
